The MFA widget should be created on the CyberArk Identity portal and the HTML should be downloaded from the portal to embed the MFA widget

The MFA widget is used by Acme Inc for second factor authentication at the time of login. The user will be prompted with the MFA setup at the user policy/app policy. The username will automatically be populated for the MFA widget.
The MFA widget internally uses the “/startAuthentication” and “/advanceAuthentication” API of CyberArk Identity to login the users to Acme Inc. The look and feel of the widget can be dynamically changed from the CyberArk Identity portal and the properties get rendered based on the widget ID.
The widget can be integrated with authentication and authorization protocols such as OIDC/SAML to generate tokens for the user. By default a one time authentication token will be returned by the widget. If the domain of Acme Inc is registered with CyberArk Identity, then the widget automatically generates cookies for Acme Inc domain.
For more information of the widget please check the documentation here.
