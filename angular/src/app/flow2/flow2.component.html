ACME UI + CyberArk Identity Sign Up/Sign In Widget

  • This demo web app simulates a Acme.com securing their end user facing website using CyberArk Sign Up/Sign In widget
  • Admin needs to map the role in CyberArk Admin portal for newly registereed user
  • ACME authenticates the identity of end users using CyberArk Identity
  • Profile update powered by CyberArk APIs
  • The backend is driven by Java spring, while Angular is used for the front-end of the application.

The following APIs are used in this web application

Profile Update

/user/UpdateProfile - Update current user information.

Logout

/Security/Logout - Logout current user.
