ACME UI + APIs

  • This demo web app simulates a customer securing their end user facing website using CyberArk Identity APIs.
  • The backend is driven by Java spring, while Angular JS is used for the front-end of the application.
  • Each user is created and stored in CyberArk Identity.

The following APIs are used in this web application

Sign up

/CDirectoryService/Signup - Create user in the cloud directory service and auto logs in

Login

/Security/StartAuthentication - The authentication package returned is a complex structure that guides the caller through the authentication process. Multiple calls to AdvanceAuthentication may be required to complete the process.

MFA

/Security/AdvanceAuthentication - Attempt to advance the state of an authentication session. When things are going smoothly, you can expect to get a 'StartNextChallenge' or 'LoginSuccess' as response.

Profile Update

/user/UpdateProfile- Change cloud user properties.

Logout

/Security/Logout - Logout current user.
