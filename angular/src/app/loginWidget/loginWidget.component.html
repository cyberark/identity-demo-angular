The signup widget should be created on the CyberArk Identity portal and the HTML should be downloaded from the portal to embed the signup widget

The signup widget internally uses the “/Signup” API of CyberArk Identity to signup and create the users.
The fields on the signup widget and the look and feel of the widget can be dynamically changed from the CyberArk Identity portal and the properties get rendered based on the widget ID.
The signup widget supports using a bearer token (or) captcha to create users.It can also be used along with Identity verification to prevent identity theft and consent management to get implicit consent of the user.
The widget can be integrated with authentication and authorization protocols such as OIDC/SAML to generate tokens for the user if auto login is enabled.
For more information of the widget please check the documentation here.

The login widget should be created on the CyberArk Identity portal and the HTML should be downloaded from the portal to embed the login widget

The login widget internally uses the “/startAuthentication” and “/advanceAuthentication” API of CyberArk Identity to login the users to Acme Inc. The user will be prompted with the MFA setup at the user policy/app policy.
The look and feel of the widget can be dynamically changed from the CyberArk Identity portal and the properties get rendered based on the widget ID.
The widget can be integrated with authentication and authorization protocols such as OIDC/SAML to generate tokens for the user. By default a one time authentication token will be returned by the widget. If the domain of Acme Inc is registered with CyberArk Identity, then the widget automatically generates cookies for Acme Inc domain.
For more information of the widget please check the documentation here.
