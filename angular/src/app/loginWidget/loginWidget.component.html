The admin of Acme Inc. creates the signup widget on the CyberArk Identity portal and downloads the HTML from the portal to embed the signup widget

The signup widget internally uses the “/Signup” API of CyberArk Identity to signup and create the users.
The signup widget’s fields and look and feel dynamically change based on the widget ID.
The signup widget supports creating users using a bearer token (or) captcha. You can use the widget alongside Identity verification to prevent identity theft and consent management.
You can integrate the widget with authentication and authorization protocols such as OIDC/SAML to generate tokens for the user if auto-login is enabled.

The admin of Acme Inc. creates the login widget on the CyberArk Identity portal and downloads the HTML from the portal to embed the login widget.

The login widget internally uses the “/StartAuthentication” and “/AdvanceAuthentication” API of CyberArk Identity to log in the users to Acme Inc. The widget prompts the user with the MFA as per the user policy/app policy.
The login widget’s fields and look and feel dynamically change based on the widget ID
You can integrate the widget with authentication and authorization protocols such as OIDC/SAML to generate tokens for the user. The widget returns a one-time authentication token by default. If Ame Inc. registers its domain with CyberArk Identity, the widget automatically generates cookies for the Acme Inc domain.
For more information on the widget, please check the documentation here.
